Module 01 — Metagenomic Analysis

FASTQ in. Taxonomy, diversity, and a report out.

This is a real, working slice of OmniOmics Cloud: it parses actual FASTQ records in your browser, computes genuine quality and composition statistics, and renders the same kind of report a researcher would pull from the full platform.

What's real vs. simulated here: Upload parsing, quality scoring, trimming, GC content, and diversity math run on the actual sequence data you give it. Taxonomic classification uses a small illustrative reference set (not SILVA/NCBI) — in production this stage hands off to Kraken2, QIIME 2, or DADA2 against a real reference database, per the platform roadmap. Use "Load demo dataset" for a synthetic 3-sample microbiome to see the full pipeline end to end.

Module 02 — Sanger Sequence Analysis

A single read, trimmed, and checked against a reference.

Paste or upload a Sanger read (FASTA, or FASTQ with quality). The module trims low-quality ends, computes real sequence stats, and aligns it against a small reference panel to flag mismatches — the same shape as an AB1 → trim → BLAST → mutation-call workflow.

What's real vs. simulated here: Trimming, GC%, length, and the mismatch alignment against the chosen reference are computed for real on whatever sequence you provide. The reference panel is a small illustrative set, not a live BLAST/GenBank search — production would call NCBI BLAST or a local BLAST+ database, per the roadmap.

Module 03 — Genome Analysis

One FASTA in, the same core stats out — whatever the kingdom.

Plant, animal, virus, fungus, or bacterial genome/assembly: the sequence-statistics layer is the same regardless of organism. Upload a FASTA, tag its kingdom for organization, and get real length, GC, N50, ORF, and codon-usage analysis.

What's real vs. simulated here: Every number on this page — length, GC%, N50, ORF count, codon usage — is computed directly from the FASTA you load, real or demo. What's not here yet: gene annotation, repeat masking, and comparative genomics against RefSeq/Ensembl — those need reference databases and tools like Prokka, AUGUSTUS, or BUSCO running on a real backend, per the platform roadmap.
